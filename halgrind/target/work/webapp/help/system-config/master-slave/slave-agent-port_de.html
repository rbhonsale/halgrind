Hudson verwendet einen TCP-Port, um mit Slave-Knoten zu kommunizieren, die über JNLP gestartet wurden. Normalerweise wird dieser Port zufällig gewählt, um Kollisionen zu vermeiden - dies erschwert jedoch die Absicherung des Systems. Wenn Sie keine JNLP-Slave-Knoten einsetzen, wird empfohlen, diesen TCP-Port zu deaktivieren. Alternativ können Sie auch einen statischen Port festlegen, um Ihre Firewall entsprechend leichter konfigurieren zu können.
