Ein "Touchstone"-Build ist ein besonderer Build, der Sie besonders frühzeitig auf Probleme in Ihrem Code hinweist, z.B. durch Ausführung essentieller Plausibilitätstests.

Wenn angewählt, führt Hudson zunächst die Touchstone-Builds aus, bevor dann alle weiteren Kombinationen gebaut werden. Sie können über einen Kombinationsfilter auch mehrere Builds als Touchstone-Build kennzeichnen. Nur wenn das Ergebnis der Touchstone-Builds die Ergebnisbedingung erfüllt, werden alle weiteren Kombinationen gebaut.
